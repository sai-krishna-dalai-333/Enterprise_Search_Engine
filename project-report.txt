Project Phase 3: Enterprise Search Engine Implementation
Architecture and Performance Evaluation Report

1. Executive Summary
This report outlines the design, implementation, and evaluation of a custom Enterprise Search Engine. The system processes a corpus of documents to construct a functional search index utilizing the Vector Space Model (VSM). By calculating Term Frequency-Inverse Document Frequency (TF-IDF) weights and evaluating query vectors via Cosine Similarity, the system successfully retrieves and ranks documents based on their relevance to multi-tiered user queries.

2. Architectural Design & Engineering
Unlike traditional flat-script implementations, this system was engineered using Domain-Driven Design (DDD) principles to ensure modularity, scalability, and type safety. The codebase is heavily reliant on Java 8+ functional programming paradigms, specifically Streams and Iterators, to optimize data transformation.

2.1 Package Structure and Components
* models Package: Contains data entities such as SearchIntent (encapsulating the headline, summary, and context of a query) and ResultNode (representing the final scored document).
* nlp (Natural Language Processing) Package: Houses the TextCleaner and the Porter stemmer. This layer is responsible for token normalization, stripping non-alphanumeric characters, removing noise via a stopword list, and reducing terms to their morphological roots.
* core Package: Contains the heavy lifting of the search engine.
  - The CorpusAnalyzer handles the ingestion of raw XML-style text files, dynamically mapping global term counts and generating the TF-IDF matrix.
  - The SearchEvaluator builds the query vectors, calculates the dot-product similarity against the document magnitudes, and uses Java Streams to sort and limit the output.

2.2 Vector Space Mathematics
* Weighting: The system dynamically calculates TF-IDF weights during the vector generation phase. The Inverse Document Frequency (IDF) is calculated as the natural logarithm of the total corpus size divided by the global term frequency.
* Normalization: To prevent document length from skewing relevance, the system calculates the Euclidean magnitude (length) of both the document and query vectors, applying exact Cosine Similarity to determine the final rank.

3. System Evaluation and Query Configuration
The system was evaluated against a standard set of manually judged relevance metrics (qrels) using varying levels of query depth (Headline only, Headline + Summary, Headline + Context).

3.1 Configuration Impact
* Baseline (Headline Only): Using only 2-3 keyword phrases results in strict but rigid matching. It operates exceptionally fast but lacks the vocabulary to overcome synonym mismatches.
* Expanded Context (Headline + Context/Narrative): By expanding the SearchIntent to include the descriptive narrative, the query vector becomes much denser. Our testing demonstrated that utilizing the maximum context configuration drastically improves the system's ability to pull relevant documents into the top 100 rankings.

3.2 Overall Performance View
The evaluation proved that the underlying TF-IDF math is executing flawlessly. The system demonstrated highly efficient Recall on specific, niche topics (such as queries regarding "Antarctica exploration" or "Journalist risks"), successfully retrieving a large percentage of all known relevant documents within just the top 100 results (e.g., retrieving 45% and 32% of all relevant documents respectively).

Conversely, the system exhibited expected limitations on broad topics containing highly frequent vocabulary (such as general queries about the "British economy"). Because the corpus consists primarily of financial news, terms like "economy" cause the Vector Space Model to retrieve false positives. This highlights the standard constraints of pure Cosine Similarity and provides a clear pathway for future enhancements, such as implementing BM25 probabilistic weighting to better penalize overly common terms.

4. Conclusion
The modular architecture of the Enterprise Search Engine provides a highly stable, extensible foundation for document retrieval. The system successfully leverages core mathematical models to rank documents accurately, and the evaluation confirms that expanding search intent with narrative context provides the most robust retrieval performance within a standard Vector Space framework.
